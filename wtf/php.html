A list of PHP weirdness...

operator precedence

PHP's operator's don't always follow the C standard. This can lead to subtle bugs, so the recommendation is to use and abuse () in expressions.

PHP code

<?php

echo 1 ? 2 : 3 ? 4 : 5;

output

4

C code

#include 

int main() {
  printf("f=%d\n", 1 ? 2 : 3 ? 4 : 5);
  return 0;
}

output

2

Calling instance methods in static context

PHP code

<?php

class A {
  public function printName() {
   echo 'I am:', get_class($this), "\n";
  }

  public function foo() {
    return A::printName();
  }
}

class B {
  public function foo() {
    A::printName();
  }
}

$a = new A();
$a->foo();

$b = new B();
$b->foo();

output

I am: A
I am: B

class A {

    parent::__construct();

new B();

Redefine private methods

PHP code

<?php

class A {
  private function foo() {
    return 'foo in A';
  }

  public function bar() {
    echo $this->foo(), "\n";
  }
}

class B extends A {
  private function foo() {
    return 'foo in B';
  }

  public function bar2() {
    echo $this->foo(), "\n";
  }
}

$b = new B();
$b->bar2();
$b->bar();

output

foo in B
foo in A
